## 2. HACKER NEWS — Show HN

**Title:** Show HN: Docify – documentation that updates itself when your code changes
**URL:** https://docifydocai.com

**Text (paste into the text field):**
I'm a solo founder. I kept seeing the same failure at every company: the code moves daily, the docs are frozen at v0.9, and nobody's job is to fix it.

Docify connects to GitHub/GitLab/Bitbucket (read-only OAuth; source is never stored). On a merged PR it reads the diff and PR context, finds the best-matching section of your existing documentation via semantic placement, rewrites only that section (versioned, restorable), and runs a quality gate (content/links/style — configurable threshold) before publishing. No duplicate files, ever.

Two other things it does: on-demand generation from repos/Jira/files/OpenAPI specs into DITA/Markdown/HTML/PDF/Word, and an "AI readiness" score — it evaluates how well retrieval systems (ChatGPT, Gemini, Claude, Copilot) can parse, chunk, and cite your pages, with concrete fixes.

Stack: React + Express + Postgres. Happy to answer anything about the placement scoring, the quality gates, or the business of selling docs automation.
